BP미디어 기사 작성 표준 v2.3

스카우트 관련 원문을 기반으로 BP미디어 기사를 일관된 구조로 생성하기 위한 단일 기준. v2.3은 v2.2를 유지한 채 검색 최적화(SEO)와 메시지 전달력 강화를 위해 Title·Subtitle 규칙을 확장한다.

버전 v2.3 상태 운영 중

목적

이 문서는 스카우트 관련 원문을 기반으로 BP미디어 기사를 일관된 구조로 생성하기 위한 단일 기준이다. v2.3은 기존 v2.2를 유지하면서 검색 최적화(SEO)와 메시지 전달력 강화를 위해 Title과 Subtitle 규칙을 확장한다.

핵심 원칙

01사실은 고정한다
02구조는 정리한다
03해석은 통제한다
04표현은 겸손하고 절제한다
05홍보는 구조로 드러낸다

출력 형식 고정

아래 순서를 반드시 유지한다.

  1. Title
  2. Subtitle
  3. Body
  4. 자료출처:
  5. Recommended Tags:

Title 규칙 v2.3 핵심 개정

기본 구조

주체 + 행위 + 사건명

SEO 확장 규칙

  • 핵심 키워드를 제목 앞부분에 배치
  • 검색 의도 기반 키워드 사용 (회의, 교육, 프로그램, 다양성·포용 등)
  • 정보 밀도로 후킹 생성 (감정 금지)

행위 규칙

  • 실제 발생한 행동만 사용 (개최, 운영, 체결 등)
  • 추상적 표현 금지 (추진, 강화, 확대 단독 사용 금지)

길이 기준

  • 28~36자 권장
  • 검색 결과에서 잘리지 않는 길이 유지

후킹 방식

  • 감정 유도 금지
  • 정보량으로 클릭 유도
  • 실제 검색 형태 반영

금지

  • 감정 표현
  • 평가
  • 미래 예측
  • 과장 표현
  • 중복 표현

Subtitle 규칙 v2.3 핵심 개정

구조 정의

부제목 = 왜 했는가 + 어떤 임팩트가 있는가

표현 방식

  • 반드시 명사형으로 종료
  • 문장형 금지
  • 설명형 금지

구성 요소

  • 목적 (왜 했는가)
  • 적용/변화 (어떤 영향이 있는가)

권장 단어

확장 / 적용 / 실행 / 기준 / 방향 / 전환 / 운영 / 구축

후킹 방식

  • “왜 읽어야 하는지”를 즉시 이해 가능하게 구성
  • 메시지를 압축하여 전달

금지 요소

  • 구조, 개요, 설명 등 추상어
  • 감정 표현
  • 평가
  • 단순 사실 반복

Body 구조 최소 4문단

1
사건 설명

주체, 시점, 장소, 행위 명확히 제시

배경

행사 목적 또는 맥락

3
전개

4
확장

운영 방식 또는 흐름 제시 (단정 금지)

확장 가능

  • 필요 시 5문단 이상 가능
  • 메시지 중복 금지

작성 기준

  • 문단당 하나의 메시지
  • 2~5문장 유지
  • 시간 또는 논리 흐름 유지
  • 설명보다 행위 중심 서술

번역 및 표기 규칙

  • 모든 내용은 원문 기반 사실만 사용
  • 임의 해석 및 창작 금지
  • 연맹명: 국문(영문) 최초 1회 병기 후 국문 사용
  • 직함 필요 시 병기
