## 기능 중심으로 보는 이유

- 실제 작업은 파일이 아니라 기능 단위로 들어옵니다.
- 하나의 기능 아래에 `모듈 / 템플릿 / API / 설정`이 함께 보여야 변경 영향 범위를 빠르게 판단할 수 있습니다.
- 다른 AI나 외부 API가 붙더라도 기능 허브만 따라가면 필요한 문맥을 좁게 읽을 수 있습니다.
